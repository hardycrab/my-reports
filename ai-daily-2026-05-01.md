# AI 动态日报 - 2026 年 05 月 01 日

## 🌍 热点速递

1. **OpenAI 发布 GPT-5.5**（4 月 23 日）- OpenAI 正式推出 GPT-5.5 模型，同步发布系统卡和安全评估报告。英国 AI 安全研究所 (AISI) 评估显示 GPT-5.5 的网络安全能力与 Claude Mythos 相当，但 GPT-5.5 已公开发布。

2. **Codex CLI 0.128.0 新增 /goal 功能**（4 月 30 日）- OpenAI 的 Codex CLI 编码智能体引入 Ralph loop 机制，用户可设置目标后让 Codex 持续循环执行直至完成或耗尽 token 预算。

3. **Bun 被 Anthropic 收购后取得重大性能突破**（4 月 30 日）- 2025 年 12 月被 Anthropic 收购的 JavaScript 运行时 Bun，在其 Zig 分支上实现了 4 倍编译性能提升，但因 Zig 项目严格的 LLM 贡献禁令无法上游合并。

4. **OpenAI 与微软进入合作伙伴关系新阶段**（4 月 27 日）- 双方宣布深化合作，OpenAI 模型、Codex 和托管智能体正式登陆 AWS。

5. **Anthropic 发布 Claude Design**（4 月 17 日）- Anthropic Labs 推出新产品，让用户与 Claude 协作创建视觉作品，包括设计、原型、幻灯片等。

## 🤖 Agent 应用

- **Codex CLI 智能体增强**：新版本通过 goals/continuation.md 和 goals/budget_limit.md 提示模板，实现目标驱动的自主循环执行能力。

- **开源编排规范 Symphony**（4 月 27 日）- OpenAI 发布开源的 Codex 编排规范，旨在标准化智能体工作流。

- **LLM 0.32a0 重大更新**（4 月 29 日）- Simon Willison 的 LLM Python 库和 CLI 工具发布重大重构版本，改进工具调用对话的 SQLite 持久化。

## 🔐 安全动态

- **GPT-5.5 网络安全能力评估** - 英国 AISI 评估显示 GPT-5.5 在发现安全漏洞方面能力与 Claude Mythos 相当，引发对公开发布模型安全边界的讨论。

- **Project Glasswing 启动**（4 月 7 日）- Anthropic 联合 AWS、Apple、Google、Microsoft、NVIDIA 等 11 家科技巨头，共同保护全球关键软件安全。

- **pip 26.1 引入依赖冷却机制**（4 月 28 日）- Python 包管理器新增 --uploaded-prior-to 参数，可指定只安装发布超过一定天数的版本，降低供应链攻击风险。

- **Zig 项目严格反 LLM 政策** - Zig 成为对 LLM 贡献政策最严格的主流开源项目之一，禁止 LLM 生成的 issue、PR 和评论。核心贡献者解释：开源项目应投资于贡献者而非代码。

## 🚀 技术迭代

- **Talkie-1930 复古语言模型** - Nick Levine、David Duvenaud 和 Alec Radford 发布基于 1931 年前英文文本训练的 13B 模型，探索知识截止对模型能力的影响。

- **pip 26.1 锁文件支持** - 新增 `pip lock` 命令，可生成完整的依赖锁定文件 (pylock.toml)，提升 Python 项目可复现性。

- **GPT-5.5 系统提示泄露** - OpenAI Codex 的 base_instructions 显示模型被禁止讨论"goblins, gremlins, raccoons, trolls, ogres, pigeons"等动物或生物。
