# AI 动态日报 - 2026 年 05 月 02 日

## 🌍 热点速递

1. **第二次 API 开放浪潮到来** - 随着大模型在生产环境的普及，平台纷纷开放 API 以接入 AI 工作流。腾讯在 OpenClaw 爆红后快速开放微信接口，让 AI 可以向微信发消息。这次开放比 15 年前的浪潮更彻底，涵盖外卖、电商、银行等日常生活服务。

2. **GPT Images 2.0 发布** - OpenAI 发布最新图像模型，文字渲染能力大幅提升，很好支持汉字生成。性能超过谷歌 Nano Banana 2 Pro，可在 ChatGPT.com 免费试用。

3. **人形机器人半马比赛** - 北京亦庄举行第二次人形机器人半马比赛，冠军成绩 50 分 26 秒，超过人类世界纪录（1 小时 02 分 52 秒）。但机器人需频繁更换电池和冷却，续航仍是瓶颈。

4. **AI 歌手占领 iTunes 榜单** - 虚拟 AI 歌手"艾迪·道尔顿"在 iTunes 单曲榜前 100 名中占据 11 席，两首歌进入前 10 名，引发 AI 与人类争夺艺术工作的讨论。

5. **Zig 语言实施严格反 AI 政策** - Zig 项目禁止使用 LLM 提交 issue、PR 和评论，认为 LLM 辅助破坏开源社区培养新贡献者的机制。

## 🤖 Agent 应用

### 编程 Agent 进展
- **Codex CLI 0.128.0** 新增 `/goal` 功能，可设置目标后持续循环执行直到完成或 token 预算耗尽
- **mini-cc** - 开源 AI 编程 Agent，类似 Claude Code，采用多语言架构，已完成 TypeScript 实现
- **EVA** - 极简 AI 编程智能体，仅需单个 Python 脚本，定位为低配版 Claude Code
- **Nezha（哪吒）** - 开源 AI 编程任务管理器，快速切换多任务，集成终端、会话管理、代码编辑、Git 等功能，大小不到 10MB

### 应用工具
- **摄像头耳机** - 华盛顿大学开发世界首个带微型摄像头的无线耳机，可与 AI 互动询问眼前事物
- **OmniVoice Studio** - 视频配音 AI 桌面应用，支持语音翻译和克隆，完全本地生成
- **Input 0** - 免费开源 macOS 语音输入工具，本地运行，支持大模型识别语音并进行文本润色

### AI 扩展能力
AI 展现"扩展神器"特性：可将 CEO 口述的设想依次扩展为战略文件→产品规格→代码原型→发布文案→公关稿，实现从想法到产品的快速转化。

## 🔐 安全动态

### 大模型安全评估
- **UK AISI 评估 GPT-5.5 网络能力** - 英国 AI 安全研究所评估发现，GPT-5.5 在发现安全漏洞方面与 Claude Mythos 相当，但 GPT-5.5 已公开发布
- **安全投入竞赛** - 评测者发现残酷事实：花费的 Token 费用越多，发现的漏洞越多，系统越安全。安全行业变得像工作量证明，投入多者胜

### 隐私保护
- **OpenAI Privacy Filter** - 本地运行的大模型，先处理敏感信息（姓名、地址、电话、密码）再发给线上模型。例如"产品发布日期是 2026 年 9 月 18 日"处理后变为"产品发布日期是 [PRIVATE_DATE]"

### 供应链安全
- **pip 26.1 新增锁文件和依赖冷却** - Python 包管理器新增 `pip lock` 命令生成锁文件，支持 `--uploaded-prior-to PXD` 选项要求包至少发布 X 天，防止恶意包快速传播
- **Claude Code 源码泄漏分析** - 泄漏源码全部由 AI 生成，质量不高，单函数达 3167 行含 486 个判断分支。预示 AI 编程流行后，代码泄露、供应链攻击将成为新常态

### 版权争议
- **大模型权重版权归属** - 有观点认为权重是计算结果而非创作，可能没有版权，因此开源大模型可能无权设置或修改许可证

## 🚀 技术迭代

### 模型进展
- **GPT Images 2.0** - OpenAI 最强图像模型，文字渲染和汉字支持大幅提升
- **Talkie-1930-13B** - 由 GPT 共同作者 Alec Radford 参与开发的复古语言模型，训练数据为 1931 年前的英文文本，探索"模型能否预测未来"等研究问题

### 开发工具
- **LLM 0.32a0** - Simon Willison 的 Python LLM 库重大重构，支持更多模型和工具调用
- **LinkAI Gateway** - 开源 AI 网关，接入主流大模型，对外提供统一 API（OpenAI 兼容）和管理后台
- **TokenTracker** - 生成本地 Token 消耗统计报表，支持 Claude Code、Codex、Cursor、Gemini、OpenClaw 等多种 Agent

### 浏览器集成
- **Chrome 原生支持技能** - 官方宣布在 Gemini 插件中支持技能（skill），即预置提示词一键完成任务，代表浏览器发展方向

### 开源生态
- **vibe-coding 应用激增** - AI 加速应用开发后，微应用变得个人化、情景化、高频化。社区呼吁为工具页面建立 RSS 订阅，便于分享和发现

## 🗞️ 阮一峰周刊摘要

### 第 394 期核心观点（2026-04-25）

**第二次 API 开放浪潮**
- 2011 年第一次 API 开放潮因平台难以盈利、数据帮助竞争对手而失败
- 2025 年下半年大模型达到临界点，AI 自动化需要调用其他平台 API
- 没有 API 的平台无法进入 AI 工作流，将被市场放弃
- 这次开放更彻底：覆盖生活服务、自然语言调用、消费者通过 AI 调用

**AI 是扩展神器**
- AI 不仅压缩信息，更善于扩展信息，填补模糊和缺失部分
- 可将一个想法不断扩展出文档、代码、产品、发布会

**工具推荐**
- WatermarkZero：Gemini 生成图片的可见水印去除工具，本地浏览器处理
- Himi Recorder：开源 Mac 录屏应用，可绕过录屏检测

**脑腐状态（Brain Rot）**
